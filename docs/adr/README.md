# Arbor Architecture Decision Records

Architecture Decision Records explain durable technical choices for Arbor. Write an ADR when a decision changes platform direction, data boundaries, model routing, security posture, deployment shape, or knowledge-source ownership.

Each ADR should stay short and use this structure:

- Status
- Context
- Decision
- Consequences
- Alternatives
