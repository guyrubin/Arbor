### Risks
- GLOBAL vs PARENT scope leak: `cardCls` (tokens.ts) is used both inside `.arbor-parent` AND in the global `.arbor-app` sapphire-glass surfaces. Re-pointing its shadow to `--shadow-md` changes BOTH; verify the glass surfaces still look right (their `--shadow-md` is blue-tinted glass, the parent's is neutral-soft). Safer: keep `cardCls` token-driven but confirm no visual regression outside `.arbor-parent`.
- SectionCard default change (icon 36->40, title text-lg->--t-md) will visibly resize EVERY existing section card app-wide, including non-redesigned screens — could look inconsistent mid-migration. Consider gating via a `dense`/`parent` prop rather than flipping the default.
- Tone-set trap: kit primitives use mint|coral|lav|yellow|pink|sky; the design's accent-coded stats/icons map to sapphire(=accent), peach(courage/focus), sky. A new StatTile/IconChip taking a PlayKit tone (clay/peach default) renders BLANK — must default to a kit tone and document this.
- RTL: design uses `inset-inline-end`/`border-inline-end` correctly; if new utilities are authored with left/right they break Hebrew mirroring. The conic-gradient dev-map ring and hero scrim are direction-agnostic, but the absolute tag pill and notification dot use `inset-inline-end` — preserve logical props.
- `.scrIn` keyed on viewKey: design re-fires the animation on every view change via React `key`. If applied to a wrapper that React doesn't remount, it fires once only. Implementer must put the animation on a `key={viewKey}` wrapper (matches design's `<div key='{{viewKey}}' class='scrIn'>`), or it silently no-ops on navigation.
- reduced-motion: the global guard zeroes animation-duration; verify `.scrIn` and `.cardh` transform inherit it (they should via the existing `.arbor-app *` rule) so no motion-sensitivity regression.
- Material Symbols trap: the design source uses `.msr` Material Symbols icons and Plus Jakarta/Baloo fonts — these are the design's own stack and are OUT OF SCOPE per locked decision #1. Do NOT import Material Symbols or the design's fonts; map every `<span class='msr'>name</span>` to the lucide-react equivalent. Easy to accidentally copy icon names as-is.
- Hue trap: design literals are `#2b7fff` sapphire / `#14225a` navy — these already equal `--arbor-clay` / `--arbor-ink` under `.arbor-parent`, so use the TOKENS, never hardcode the hex (the repo has a no-raw-hex sweep discipline; 344-literal history). New utilities must reference var(--arbor-*) only.
